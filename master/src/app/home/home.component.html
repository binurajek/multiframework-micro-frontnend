Multi-Framework Micro Frontend Architecture

This dashboard demonstrates a robust Micro Frontend (MFE) implementation using Module Federation. It seamlessly integrates independent applications built with different UI frameworks into a single "Shell" environment.

Angular MFE

A native Angular micro-frontend. It shares core dependencies (like @angular/core) with the Shell to minimize bundle size while maintaining isolation.

React MFE

A React application integrated into this Angular Shell. We use a generic MfeWrapperComponent to mount the React root and handle its lifecycle events and error states.

How It Works

1. Module Federation

Webpack's Module Federation Plugin allows us to load remote modules (Remote Entry files) at runtime. The Shell doesn't need to know the code at build time.

2. MFE Wrapper Strategy

For non-Angular remotes (like React or Vue), we use a wrapper component that:
• Loads the remote module dynamically.
• Creates a host element in the DOM.
• Bootstraps the remote framework (e.g., ReactDOM.createRoot).
• Cleanly unmounts it on destruction to prevent memory leaks.

3. Shared Dependencies

We configure shared libraries in webpack.config.js. Compatible versions are shared (singleton), preventing us from loading Angular/React multiple times if possible, or handling version mismatches gracefully.

4. Fallback & Error Handling

If a remote is down, our global error handling middleware intercepts the failure and displays a user-friendly "Retry" interface instead of crashing the application.
